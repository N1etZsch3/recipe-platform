# 文档规划

本目录用于沉淀后端功能文档，覆盖业务功能、接口边界与系统能力。当前按模块拆分，避免单文档过长，便于维护与更新。

## 1. 文档清单（已落地）

1. `00-后端功能总览.md`：模块划分、角色权限、功能地图与关键流程。
2. `02-用户与认证.md`：验证码、注册登录、资料与密码、退出登录、Token 黑名单。
3. `03-菜谱与分类.md`：菜谱生命周期、发布/修改/下架、分页与详情、分类管理。
4. `04-互动与评论.md`：收藏、评论/回复、点赞、信息中心接口。
5. `05-社交与私信.md`：关注、粉丝、私信聊天、会话与已读。
6. `06-管理后台.md`：后台登录、仪表盘、审核、用户/分类/评论/日志。
7. `07-系统能力与基础设施.md`：安全、缓存、限流、幂等、WebSocket、文件上传、异常。

## 2. 预留/可选文档（视需求补充）

- `08-接口索引.md`：所有 API 汇总索引（方法/路径/权限）。
- `09-数据字典.md`：关键表字段与枚举、状态码映射。
- `10-运维与部署.md`：环境变量、部署方式、运行与排障。

## 3. 维护规范

- 文档与代码保持一致：新增/修改接口时同步更新对应模块文档。
- 规则：先更新模块文档，再补充索引或数据字典。
